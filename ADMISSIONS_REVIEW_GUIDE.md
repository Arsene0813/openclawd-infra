## What to Review First

1. `README.md`  
Main project overview, current system capabilities, example workflows, and running instructions.

2. `eval/eval_report.md`  
Scenario-based evaluation showing whether the memory layer can retrieve current facts, overwrite outdated facts, separate product entities, filter non-facts, and refuse unsupported queries.

3. `retail_ops/README.md`  
A SQL-based extension showing how fragmented multi-store retail metrics can be structured, checked for data quality, screened for comparability, and converted into conservative operational memory.

4. `retail_ops/outputs/comparability_report.md`  
A short output report demonstrating that the system does not force direct comparison when stores have missing data, low order volume, or different operating contexts.

5. `case_studies/from_livestream_to_retail_decision_support.md`  
A short case study explaining how the same memory architecture can extend from livestream product memory to store-level retail decision support.

6. `PROJECT_SUMMARY_FOR_ADMISSIONS.md`  
A concise summary of the project motivation, problem, solution, example, and relevance to my intended studies.
